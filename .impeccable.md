## Design Context

### Users

Technical developers debugging web performance. They arrive with a URL and want to leave with a clear understanding of what's wrong and what to fix first. The interface should evoke **confidence** (the data is accurate) and **clarity** (the next step is obvious).

### Brand Personality

**Technical, Warm, Actionable.**
Expert knowledge delivered in an approachable, earthy way with clear next steps. Not cold or sterile like a dashboard tool, not playful like a consumer app. The Vecton aesthetic is its own thing: warm beige canvas, orange energy, monospace precision.

### Aesthetic Direction

- **Visual tone:** Warm technical. Beige canvas (#E8E9D7) with dotted grid texture. Orange (#FF5631) as the energy accent. Near-black (#24140D) text at varying opacities for hierarchy. No shadows, flat and clean with transparency layering.
- **Reference:** The current Vecton aesthetic is the direction. Refine, don't reinvent. Not Vercel-cold, not GTmetrix-dense, not generic dashboard.
- **Anti-references:** Generic SaaS dashboards (blue + gray), cramped data tables with no breathing room, dark mode for its own sake.
- **Theme:** Light mode only. Single cohesive palette.

### Design Principles

1. **Data first, decoration second.** Every visual element must serve the metric. If a pattern or animation doesn't help the developer understand performance faster, remove it.
2. **Warm precision.** Use monospace (JetBrains Mono) for data and sans-serif (Instrument Sans) for narrative. The contrast creates hierarchy without needing size variation.
3. **Opacity as depth.** Layer information using opacity variants (vecton-dark/5 through /80) instead of shadows or elevation. This keeps the flat aesthetic while creating readable depth.
4. **Pills over prose.** Badges, pills, and compact indicators over long text labels. Status should be scannable at a glance (RatingPill, SeverityPill, tech stack chips).
5. **Progressive disclosure.** Show the score and rating first, expand into issues, then drill into code. Never overwhelm with everything at once (accordion pattern in IssueCard).

### Design Tokens

#### Colors

| Token            | Hex     | Usage                               |
| ---------------- | ------- | ----------------------------------- |
| vecton-orange    | #FF5631 | Primary accent, CTAs, active states |
| vecton-purple    | #7E43BA | Secondary accent                    |
| vecton-light     | #F9FAF6 | Card backgrounds                    |
| vecton-dark      | #24140D | Text (at /50-/80 opacity)           |
| vecton-beige     | #E8E9D7 | Page background                     |
| vital-good       | #0cce6b | Good rating                         |
| vital-needs-work | #ffa400 | Needs improvement                   |
| vital-poor       | #ff4e42 | Poor rating                         |

#### Typography

| Role         | Font            | Weight | Size                             |
| ------------ | --------------- | ------ | -------------------------------- |
| Body         | Instrument Sans | 400    | 14px (text-sm)                   |
| Labels       | Instrument Sans | 400    | 11-12px uppercase tracking-wider |
| Data/Metrics | JetBrains Mono  | 400    | varies                           |
| Code blocks  | JetBrains Mono  | 400    | 10-12px                          |

#### Spacing

- Cards: p-3 (compact), p-4 (standard), p-6 (hero)
- Grid gaps: gap-2 (tight), gap-3 (standard), gap-4 (sections)
- Section spacing: space-y-4 (within), space-y-6 (between sections)

#### Borders & Radius

- Cards: rounded-lg, border border-vecton-dark/10
- Pills: rounded-full (chips), rounded (badges)
- No shadows. Ever.

#### Animations

- Page entry: fade-up 0.6s ease-out
- Expand/collapse: 200ms grid transition
- Score gauge: 1.5s stroke-dashoffset
- Progress: 1.5s pulse (active stages)
- Respect prefers-reduced-motion

### Accessibility

- WCAG AA compliance
- Reduced motion support via @media query
- Uppercase tracking-wider labels for scanability
- Sufficient contrast on beige background
